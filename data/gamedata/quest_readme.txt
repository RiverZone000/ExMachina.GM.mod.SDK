	Subscribe: "by_request" - по запросу игрока, "automatic" - по желанию системы (при первой возможности), 
		"no" - не подписывать (это не означает, что квест нельзя выполнить)
	Нет, не так. 
	Automatic: true|false, по умолчанию false. И никаких no.
		
	SubQuestCondition может быть "and", "or", "xor" или "chain"  ВНИМАНИЕ: chain временно отменен!
		chain - то же самое, что and, но отличается порядок подписки:
		если and то при подписке на квест присходит подписка на все подквесты (далее рекурсивно)
		если or, то подписки на подквесты не происходит
		если xor, то происходит подписка на все подквесты, при этом при выполнении одного из них все остальные проваливаются.
		если chain, то происходит подписка на первый квест, при его выполнении на второй и т.д.
		Вышеописанное поведение - это поведение по умолчанию, его можно изменить (только как? ;)
		Поведение по умолчанию можно изменить только для корневых квестов.
		Или так: для некорневых его тоже можно изменить, но оно не отменяет поведения, навязанного SubQuestCondition
		
		
		Так... квест может быть дан, если произошло одно из следующий событий:
			1. Произошла подписка на какие-то квесты (и/или)
			2. Выполнены какие-то квесты (и/или)
			
		PrecedingQuests - список квестов, от которых зависит взятие данного квеста, разделенный пробелами
		
		Соответственно атрибут ConditionToGive выглядит так:
			ConditionToGive="<Operator> <EventType>"
			EventType	= taken|complete|failed
			Operator	= all|any
			Если ConditionToGive не указан, то квест изначально может быть взят (то есть тут все зависит уже от Subscribe)
		
	CheckAll = yes|no
		Если yes, то для выяснения статуса квеста нужно выполнить или провалить ВСЕ его подквесты
			

	Функции событий: можно указать функции, которые будут вызываться при изменении статуса квеста. Для этого
		используются следующие атрибуты:	
		OnComplete
		OnFail
		OnTake
		OnCanBeGiven
		Значения атрибутов - имена скриптовых функций. Функции долюны быть описаны в файле QuestStates.lua.

	Секция MutuallyExclusives:
